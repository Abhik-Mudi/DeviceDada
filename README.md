*Prompt: Create a website named "DeviceDada"*

I want to build a website named *"DeviceDada"*. The purpose of this website is to help normal people choose the right mobile or electronic device like phone, computer, laptop, tab, etc. When we buy these devices, most people don’t understand technical terms like Antutu score, UI, display type, camera MP, processor, OIS, and other specs. So this website will guide users step-by-step to find the best device for them.

*Website Flow & Features:*

1. *Homepage*
   - Show the "DeviceDada" logo.
   - First, ask the user what category of device they want to buy. Give options like: Laptop, Phone, Computer, Tab, etc.

2. *Brand Selection*
   - After selecting the category, show a list of all major companies/brands for that device.
   - Ask: "Which companies are you interested in?" 
   - User can select one or multiple companies. This is completely personal choice.

3. *Budget Selection*
   - Give an option to set/select budget range. 

4. *Usage Purpose*
   - Ask the user: "What is the main purpose of buying this device?" 
   - Options: 
     1. Gaming 
     2. Camera 
     3. Processor/Performance 
     4. All-rounder 
     5. Other etc.

5. *User Type*
   - Ask: "Who are you?" 
   - Options:
     1. School student 
     2. College student 
     3. Married man/woman 
     4. Others

6. *Technical Questions with Explanations*
   - Since most users don’t know terms like Antutu score, UI, display, camera MP, processor, OIS, etc., ask questions about each spec but also explain what each term means in simple words.
   - Example:
     *Question 1: Do you need OIS in your camera?* 
     Options: Yes / No 
     _(Explanation: OIS means Optical Image Stabilization. It keeps your camera video and photos stable and less shaky.)_
     
     *Question 2: Which processor do you want?* 
     Options: Show only those processors that companies selected by the user offer within the user’s budget. Example: Snapdragon 7 Gen 2, Snapdragon 8 Gen 3, Dimensity 9200, etc.
     _(Explanation: Processor is the brain of the phone. A better processor means faster performance and better gaming.)_
     
   - Similarly ask about: Antutu score needed, Display type, Front camera MP, Back camera MP, Battery, Charging speed, UI, etc. 
   - Every technical question must have a short, easy explanation of what that feature does.
   - All options shown must be filtered by the user’s selected budget and brands. Don’t show out-of-budget options.

7. *Comparison Page*
   - After all questions, show devices that closely match the user’s needs and budget.
   - If user selected multiple brands, e.g. Samsung and Vivo, then show the best matching phone from each brand side-by-side with photos.
   - Compare them neutrally feature-by-feature: Pros, Cons, what one has that the other doesn’t.
   - Also show public reviews and ratings side-by-side.
   - Example: If the result is Samsung A35 5G vs Vivo T4R, compare all specs, then let the user decide.

8. *Price & Availability*
   - Once the user likes a phone, show current live prices and availability from sites like Flipkart, Amazon, and other apps.
   - Prices must always be current/real-time. Not old prices.

9. *Auto-Update Database*
   - Since new phones and devices launch very often, the website database must always stay updated with the latest models. New launches should be added automatically or regularly.

10. *Additional Features You Can Add*
    - You may suggest and add more good features to make the website better, like: EMI calculator, expert score, long-term review prediction, 5G band checker, service center availability by pincode, etc.

*Important:* Keep the UX simple so that non-technical people can use it easily. The main goal is: a normal person who doesn’t know specs can still buy the perfect device by just answering simple questions.
